# MT4 ストラテジーテスター 実行手順

FX TF MT4（FXTF-MT4）で `FXTF_TickScalper.mq4` をバックテストする手順です。

## 1. ファイル配置

1. MT4を起動 → メニュー `ファイル` → `データフォルダを開く`
2. 開いたフォルダ内の `MQL4\Experts\` に **`FXTF_TickScalper.mq4`** を配置
3. MT4の「ナビゲータ」→「エキスパートアドバイザ」で右クリック →「更新」

コンパイルエラーが出なければ、緑色のアイコンで EA が表示されます。

## 2. ヒストリカルデータの準備

1. MT4メニュー `ツール` → `オプション` → `チャート` タブ
   - 「ヒストリー内の最大バー数」: **9999999999**
   - 「チャートの最大バー数」: **9999999999**
2. MT4メニュー `ツール` → `ヒストリーセンター` （F2）
   - `USDJPY` → `1分足 (M1)` を選択 → 「ダウンロード」
3. ティック精度を最大化するなら、Dukascopy 等から **tick data** を取得して
   [TickStory](https://www.tickstory.com/) または **TDS2** でMT4に投入。
   ※FXTF-MT4のデフォルトティックは生成精度が低め。

## 3. ストラテジーテスター設定

メニュー `表示` → `ストラテジーテスター` （Ctrl+R）

| 項目 | 推奨値 |
|---|---|
| Expert Advisor | `FXTF_TickScalper` |
| 通貨ペア | **USDJPY** |
| モデル | **すべてのティック (最も正確)** ← 必須 |
| 期間 | 1分足 (M1) |
| スプレッド | **現在値** または 2 (0.2pip) |
| 使用バー | 過去3〜6ヶ月 |
| 初期証拠金 | 100,000円（証拠金約1万円で10,000通貨運用想定）|
| レバレッジ | 25:1 (国内規制) または FXTF 設定値 |

## 4. EAパラメータ

「エキスパート設定」→「パラメータの入力」タブ:

### FXTF-MT4 のロット換算に注意
FXTF-MT4 のロット定義は通常 **1ロット = 10,000通貨**（ミニ口座仕様）。
口座により異なるので、MT4上で「気配値表示」→ 銘柄右クリック→「仕様」で
**契約サイズ (Contract size)** を確認してください。

| 契約サイズ | ランク1上限(1万通貨) にするロット |
|---|---|
| 10,000 (FXTF標準) | `InpLots = 1.0` |
| 1,000 (FXTF GX Micro) | `InpLots = 10.0` |
| 100,000 (海外標準) | `InpLots = 0.1` |

### 最適化済み推奨値

```
InpLots          = 1.0    # 契約サイズ10,000前提。要確認
InpMagicNumber   = 77001
InpSlippage      = 3

InpStopPip       = 2.5    # 損切幅
InpPeakActivate  = 1.8    # トレール発動ピーク
InpTrailGap      = 1.2    # トレール幅
InpTimeoutSec    = 60

InpTriggerWindow = 5
InpTriggerHits   = 3
InpEMAFast       = 20
InpEMASlow       = 60

InpMaxUnits      = 10000  # ランク1超過ブロック (ガードレール)
```

## 5. 実行とレポート確認

「スタート」ボタン → 完了後、下部タブ:

- **結果**: 個別トレード履歴（ENTRY/EXIT[STOP|TRAIL|TIMEOUT] のログ）
- **グラフ**: 資産曲線
- **レポート**: 勝率、期待値、PF、最大DD

**確認すべき指標**:
| 指標 | 期待値（シミュレータOOS） |
|---|---:|
| 勝率 | 65〜80% |
| プロフィットファクタ | 1.3〜2.0 |
| 期待値(1取引) | +1.0〜+1.3 pip |
| 最大DD | 30〜50 pip |

## 6. 最適化（パラメータスイープ）

ストラテジーテスター「最適化」タブを有効化 → EAパラメータ画面で
各変数の「スタート」「ステップ」「ストップ」を設定:

```
InpStopPip       start=1.5  step=0.5  stop=3.5
InpPeakActivate  start=1.0  step=0.3  stop=2.5
InpTrailGap      start=0.5  step=0.3  stop=2.0
```

MT4内蔵のGenetic Algorithmで数時間〜1日で収束。

## 7. 注意事項

### FXTF-MT4 特有の挙動
- **建玉連動手数料はMT4側で自動計算されない**。
  ランク1枠内なら0円だが、ロットを大きくした場合の検証時は手動で差し引く必要あり。
- **EAは両建て可能**（MT4標準仕様）。同方向建玉を増やすとランク1超過する恐れあり。
  → `InpMaxUnits = 10000` のガードで自動ブロックするよう実装済み。
- **土日跨ぎ**の3倍スワップは USD/JPY 買いではマイナス(売り方は貰える)。
  超高速取引なのでポジションは秒〜分で閉じるため影響は軽微。

### バックテスト結果の読み方
シミュレータ理論値との乖離が大きい場合:
1. **スプレッド拡大の影響** — 「現在値」設定だと固定になるので、
   tick dataに実スプレッドを含めたデータが必要
2. **スリッページ** — `InpSlippage` を 5〜10 に上げて現実に近づける
3. **約定拒否** — MT4テスターでは再現されないが、実ライブでは発生

### 本番運用前のチェックリスト
- [ ] バックテストで PF > 1.3 / 最大DD < 50pip を確認
- [ ] デモ口座で1週間フォワード運用（同じ結果か）
- [ ] ランク1ガード（InpMaxUnits）が機能しているか建玉合計をログで確認
- [ ] VPS配置 (レイテンシ < 50ms 推奨・FXTF東京リージョン)
- [ ] 週末のスワップとブリッジイベント（指標前後）の挙動確認

## 8. トラブルシューティング

| 症状 | 原因と対策 |
|---|---|
| エントリしない | `InpVerbose=true` にして Dir= の出力を確認。EMAがまだ計算されていない可能性 |
| 「OrderSend failed err=130」| ストップレベル違反（このEAはSL/TP=0固定なので通常出ない） |
| 「err=131」| ロット単位違反。契約サイズを確認 |
| 「err=134」| 証拠金不足。初期証拠金を増やす |
| 取引が多すぎる | `InpTriggerHits=4` に上げて厳格化 |
| TRAIL利確が少ない | `InpPeakActivate=1.2` に下げて早期発動 |
| パラメータが文字化け | ASCII版に修正済み。`パラメータ日本語解説.md` を参照 |
| モデリング品質が低い(24%等) | 次節「9. 高品質tickデータ」を参照 |

## 9. 高品質tickデータの取得（モデリング品質99%化）

MT4の「ヒストリーセンター」(F2)が提供するのは **M1足の始値/高値/安値/終値のみ** で、
実際のtick単位の価格推移は含まれません。そのため全ティックモードを選んでも
**M1 OHLCから疑似tickが生成される** だけで、モデリング品質は25%前後に留まります。

99%を目指すには、**外部の真tickデータ**が必要です。

### 9-1. 無料経路: Dukascopy + TickStory Lite (推奨)

Dukascopy銀行は過去tickデータを無償公開しており、それをMT4に投入します。

**手順**:

1. **TickStory Lite をDL**:
   ```bash
   open https://www.tickstory.com/
   ```
   Windows版のみ。Mac では Wine / PlayOnMac / Parallels 経由で起動

2. **TickStoryでDukascopyデータをDL**:
   - 起動 → 左ペインで `Dukascopy → USDJPY` 選択
   - 期間指定: 過去6ヶ月〜1年
   - 右クリック →「Download」→完了まで待機（数十分〜数時間）

3. **MT4へエクスポート**:
   - TickStoryで USDJPY 右クリック →「Export to MT4」
   - MT4インストールパスを指定
   - FXTFの銘柄名は `USDJPY-cd` なので、エクスポート時に銘柄名を上書き設定

4. **MT4で確認**:
   - F2（ヒストリーセンター）で USDJPY-cd を開く → データ量が膨大になっている
   - ストラテジーテスターで再実行 → モデリング品質99%表示を確認

### 9-2. 有料経路: TickDataSuite (TDS2)

TickStoryより高機能で、**可変スプレッド・スワップも再現**できる有料ツール。
- 料金: 約$97買い切り
- MT4と統合され、F2ではなくTDS2側でテスト実行
- FXTF実環境に最も近いバックテストが可能

```bash
open https://eareview.net/tick-data-suite
```

### 9-3. Mac環境での課題

TickStory/TDS2 はいずれも Windows 専用。Mac でのオプション:

| 方法 | コスト | 推奨度 |
|---|---|---|
| **Parallels Desktop + Windows 11** | 有料 | ★★★ (最安定) |
| **UTM + Windows (Apple Silicon)** | 無料 | ★★ |
| **PlayOnMac + Wine** で TickStory | 無料 | ★ (動作不安定) |
| **Macを諦めてWindows PCで準備**→.histだけMacに持ち込み | – | ★★★ |

最も簡単なのは **別のWindows PCでtickデータを準備**し、生成された `.hst` ファイルを
Macの MT4 データフォルダにコピーする方法です:

```bash
# Windows側で作成される.hstの場所
# C:\Users\<user>\AppData\Roaming\MetaQuotes\Terminal\<hash>\history\<server>\USDJPY-cd1.hst (M1)
# USDJPY-cd5.hst (M5), USDJPY-cd15.hst (M15), ...

# Mac側のコピー先
~/Library/Application\ Support/net.metaquotes.wine.metatrader4/drive_c/Program\ Files\ \(x86\)/MetaTrader\ 4/history/<server>/
```

### 9-4. 品質を上げずに信頼性を上げる代替策

tick dataの用意がすぐできない場合、以下の代替:

1. **デモ口座で2週間フォワードテスト**:
   - 実ブローカーの実tickで動く = モデリング品質100%相当
   - 2週間で100〜500取引の統計が取れる
   - 結果がバックテストと大きく乖離しなければ実運用可

2. **複数期間でのバックテスト**:
   - 同じ24.95%品質でも、3ヶ月/6ヶ月/1年 と期間を変えて再実行
   - 全期間でPF>1.0なら戦略の頑健性は確認できる
   - 特定期間だけ勝ってる=カーブフィット

3. **複数通貨でのバックテスト**:
   - USDJPY-cd / EURJPY-cd / EURUSD-cd (いずれもランク1上限10,000通貨)
   - 複数通貨で勝てる=戦略が相場特性に依存していない証拠

### 9-5. 現在のバックテスト結果(品質25%)の解釈

**+42%純益 / PF 1.18** という結果は、疑似tickでの近似値です。実際には:

- **楽観シナリオ(tick品質99%)**: 同程度またはやや改善(M1 OHLC近似は平均的)
- **悲観シナリオ(実ライブ)**: スリッページ・約定拒否により **PF 1.0〜1.1** に劣化する可能性

→ **まずはデモ口座で2週間回す** のが最も堅実な次ステップです。
