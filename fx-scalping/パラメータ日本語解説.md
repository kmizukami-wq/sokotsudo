# FXTF_TickScalper パラメータ 日本語解説

MT4内のパラメータ表示はASCIIに変更しました。以下はそれぞれの変数の日本語解説と調整ガイドラインです。

## 基本設定

### `InpLots` — ロット数
- **既定値**: `0.01` (= 100通貨、初期安全値)
- **意味**: MT4で発注する際のロット数
- **FXTF-MT4（契約サイズ 10,000通貨）での換算**:
  | InpLots | 通貨量 | ランク1との比 | 用途 |
  |---:|---:|---:|---|
  | **`0.01`** | **100通貨** | **1%** | **本番最小テスト / 既定値** |
  | `0.1` | 1,000通貨 | 10% | 初期デモ〜少額本番 |
  | `0.5` | 5,000通貨 | 50% | Tier2上限（GBPUSD/AUDUSD） |
  | `1.0` | 10,000通貨 | 100% | Tier1満額（ランク1上限） |
- **FXTF-MT4 Micro（契約サイズ 1,000通貨）の場合**:
  - 0.1 = 100通貨 / 1.0 = 1,000通貨 / 10.0 = 10,000通貨
- **海外MT4（契約サイズ 100,000通貨）の場合**:
  - 0.001 = 100通貨 / 0.01 = 1,000通貨 / 0.1 = 10,000通貨
- **確認方法**: MT4「気配値表示」→ USDJPY-cd 右クリック →「仕様」→「契約サイズ」
- **推奨ロードマップ**:
  - Week 1: `0.01` (100通貨) で慣らし運転
  - Week 2-3: `0.1` (1,000通貨) に拡大
  - Week 4以降: PF確認後 `1.0` (Tier1) / `0.5` (Tier2) へ

### `InpMagicNumber` — マジックナンバー
- **既定値**: `0` (= 通貨ペア名から自動生成)
- **意味**: このEAが発注した注文を識別する番号。複数チャートで走らせても衝突しないように
- **動作**:
  - `0` (既定): Symbol()から77000〜85999の範囲で自動計算。**多通貨並走時の最適設定**
  - `1` 以上: その値を固定使用（デモで自分の好きな番号を割当てたい時）
- **自動生成時の各通貨のマジック**（参考）:
  | 通貨 | FXTF標準 | FXTF-cd形式 |
  |---|---:|---:|
  | USDJPY | 79901 | 79873 |
  | EURJPY | 85249 | 85877 |
  | GBPJPY | 83070 | 80888 |
  | AUDJPY | 81571 | 80491 |
  | EURUSD | 77892 | 81490 |
  | GBPUSD | 84713 | 82853 |
  | AUDUSD | 83214 | 85104 |
- EA起動時にエキスパートタブに `Magic=XXXXX (auto)` と出力されるので確認可能

### `InpSlippage` — 許容スリッページ
- **既定値**: `3` (point単位)
- **意味**: 発注価格と約定価格の許容誤差。3 point = 0.3pip
- **調整**: 約定拒否が多発したら5〜10に上げる

## Exit Params（退出パラメータ）

### `InpStopPip` — 損切幅
- **既定値**: `2.5` (pip)
- **意味**: この値まで含み損が広がったら即損切
- **調整**:
  - 小さく(1.5-2.0): 負けが少なく抑えられるが勝率低下
  - 大きく(3.0-4.0): 勝率上がるがたまに大損
  - **最適化結果は2.5**

### `InpPeakActivate` — トレール発動ピーク
- **既定値**: `1.8` (pip)
- **意味**: 含み益がこの値に達したら「トレール退出」機能が有効化
- **調整**:
  - 小さく(1.0): 早めに利確開始(小さな利確多め)
  - 大きく(2.5): じっくり利益を伸ばしてから利確

### `InpTrailGap` — トレール幅
- **既定値**: `1.2` (pip)
- **意味**: ピーク利益からこの幅だけ下落したら利確
- **動作例**: peak=3.0p の状態から pip=3.0-1.2=1.8p まで戻ったら利確
- **調整**:
  - 小さく(0.5): 利確が早い。小さな揺れで切ってしまう
  - 大きく(2.0): トレンドに乗って大きく取れるが、戻りで利益減少

### `InpTimeoutSec` — 最大保有時間
- **既定値**: `60` (秒)
- **意味**: この秒数経過したら強制クローズ(レンジ相場対策)
- **調整**: 30〜120秒の範囲で調整可。1分足戦略なのでデフォルト推奨

## Entry Params（エントリパラメータ）

### `InpTriggerWindow` — トリガ評価tick数
- **既定値**: `5`
- **意味**: 直近Nティックを見て方向一致を評価

### `InpTriggerHits` — 必要な順方向tick数
- **既定値**: `3`
- **意味**: `InpTriggerWindow` 個のうち、順方向(BUYならup、SELLならdown)がN個以上あればエントリ
- **調整**:
  - 小さく(2): エントリ回数が増えるが質低下
  - 大きく(4): 厳選されて精度上がるが機会減少

### `InpEMAFast` / `InpEMASlow` — 方向判定EMA
- **既定値**: `20` / `60`
- **意味**: M1足で EMA(20) > EMA(60) なら買い方向のみ、逆ならSell方向のみでエントリ
- **調整**: 通常触らない。テクニカル分析に詳しければ調整可

## Risk Guard（リスクガード）

### `InpMaxUnits` — ランク1上限
- **既定値**: `10000`
- **意味**: 同方向の建玉＋未約定発注の合計がこの通貨数を超える新規発注をブロック
- **FX TFランク1**: USD/JPY は10,000通貨まで手数料0円
- **変更**: 他通貨ペアで使うときは表に従って変更
  | 通貨ペア | ランク1上限 |
  |---|---:|
  | USD/JPY, EUR/JPY, EUR/USD, AUD/JPY, GBP/JPY | 10,000 |
  | GBP/USD, AUD/USD | 5,000 |
  | NZD/JPY, CAD/JPY, USD/CAD, MXN/JPY | 3,000 |
  | ZAR/JPY, その他マイナー | 1,000 |

### `InpMaxSpreadPoints` — 許容最大スプレッド
- **既定値**: `2` (point = 0.2pip)
- **意味**: 現在のスプレッドがこれを超えていたらエントリしない
- **重要**: 指標発表・早朝の広スプレッド帯から自動防御する要の機能
- **調整**:
  - 厳しく(1): 0.1pip超で停止。取引機会減るが完全防御
  - 緩く(5): 0.5pipまで許容。取引機会は増えるが時々広スプレッドで約定

### `InpSkipTokyoMorn` — 日本時間6-9時除外
- **既定値**: `false` (新方針では `InpTradeOnlyEAHrs` で包括するため無効化)
- **意味**: この時間帯(JST 06:00-09:00)は薄商いでスプレッドが広がりやすいので取引停止
- **注意**: `InpTradeOnlyEAHrs=true` にしているとこの設定は不要（カスタム時間帯で包括）

### `InpSkipNYNewsHrs` — 日本時間21-24時除外
- **既定値**: `false` (新方針ではNY時間も取引する)
- **意味**: NY開場＋米指標が集中する時間帯(JST 21:00-24:00)を除外
- **注意**: 新運用では NY時間も取引対象。動的スプレッドゲート(`InpMaxSpreadPoints`)で指標時の急拡大を自動防御

### `InpJSTFromBroker` — JSTとブローカー時刻の差
- **既定値**: `7`
- **意味**: MT4サーバー時刻に何時間足すとJSTになるか
- **重要**: **冬時間=7 / 夏時間=6**
  - FXTFの夏時間切替: 3月最終日曜日 ～ 11月第1日曜日(米国準拠の場合)
  - 「ログのTimeCurrent()が実際の日本時間から何時間ずれているか」で判定
- **確認方法**: MT4で`PrintFormat("broker=%s", TimeToStr(TimeCurrent()));`を実行。
  出力時刻とPC時計のJSTを比較

### `InpTradeOnlyEAHrs` — カスタム取引時間帯を有効化
- **既定値**: `true` (新運用方針で有効化)
- **意味**: trueにすると、`InpStartHour`〜`InpEndHour` の時間帯でのみ取引
- **深夜跨ぎ対応**: StartHour > EndHour の場合、翌日EndHourまでを含む
  - 例: Start=9, End=3 → 9:00〜翌3:00 (18時間稼働)

### `InpStartHour` / `InpEndHour` — 取引時間帯
- **既定値**: `9` / `3`
- **運用例**:
  - `9` / `3` (デフォルト): JST 9:00〜翌3:00 稼働（**18時間稼働/6時間休み**）
  - `9` / `15`: 東京時間のみ (6時間)
  - `15` / `24`: ロンドン+NY前半のみ
  - `0` / `24`: 24時間稼働
- **休止時間(3:00〜9:00)の意義**:
  - 早朝の流動性低下・スプレッド拡大帯を回避
  - サマータイム移行時の不安定期を回避
  - EA監視・メンテナンスの空白時間

### `InpVerbose` — 詳細ログ
- **既定値**: `false`
- **意味**: trueにすると全ティックの判定ログをExpertタブに出力
- **注意**: バックテストではONにすると動作が遅くなる。ライブでは診断時のみON

## チューニングの優先順位

実運用前に調整するなら、優先度順に:

1. **`InpLots`** — 契約サイズに合わせる(これ間違えるとランク1超過で手数料発生)
2. **`InpJSTFromBroker`** — 冬/夏で切替
3. **`InpMaxSpreadPoints`** — ライブで気配値観察してから調整
4. **`InpStopPip` / `InpPeakActivate` / `InpTrailGap`** — MT4のGA最適化で自動チューニング

## MT4最適化（GA）の推奨レンジ

ストラテジーテスターの「最適化」を有効化し、パラメータ画面で:

| 変数 | Start | Step | Stop |
|---|---:|---:|---:|
| InpStopPip | 1.5 | 0.5 | 3.5 |
| InpPeakActivate | 1.0 | 0.3 | 2.5 |
| InpTrailGap | 0.5 | 0.3 | 2.0 |
| InpTriggerHits | 3 | 1 | 4 |

→ 約200〜300通り × シード1でGenetic Algorithm → 数時間で収束
